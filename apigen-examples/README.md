## Example Schema

All examples are generated from the same e-commerce database schema:

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  categories  │     │   products   │     │   reviews    │
├──────────────┤     ├──────────────┤     ├──────────────┤
│ id           │◄────│ category_id  │     │ id           │
│ name         │     │ id           │◄────│ product_id   │
│ description  │     │ name         │     │ customer_id  │──►
│ activo       │     │ description  │     │ rating       │
└──────────────┘     │ price        │     │ comment      │
                     │ stock        │     │ activo       │
                     │ sku (unique) │     └──────────────┘
                     │ image_url    │
                     │ activo       │
                     └──────────────┘
                            │
                            ▼
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│  customers   │     │    orders    │     │ order_items  │
├──────────────┤     ├──────────────┤     ├──────────────┤
│ id           │◄────│ customer_id  │     │ id           │
│ name         │     │ id           │◄────│ order_id     │
│ email        │     │ order_date   │     │ product_id   │──►
│ phone        │     │ status       │     │ quantity     │
│ address      │     │ total        │     │ unit_price   │
│ activo       │     │ activo       │     │ activo       │
└──────────────┘     └──────────────┘     └──────────────┘
```

## Features Demonstrated

Each example showcases these APiGen features:

- **CRUD Operations** - Create, Read, Update, Delete
- **Pagination** - Page-based pagination with sorting
- **Soft Delete** - Logical deletion via `activo` flag
- **Validation** - Input validation with custom messages
- **Relationships** - Foreign keys and lazy loading
- **Auditing** - Created/Updated timestamps and user tracking
- **API Documentation** - Swagger/OpenAPI integration
- **Docker Support** - Ready-to-deploy containers

## Running the Java Example

```bash
cd java/

# Using Docker
docker-compose up -d

# Or using Gradle
./gradlew bootRun
```

Access the API at: http://localhost:8080/api
Swagger UI at: http://localhost:8080/swagger-ui.html

## Architecture Pattern

All generated APIs follow a clean layered architecture:

```
┌─────────────────────────────────────────────┐
│              Controllers/Routers            │  ← HTTP Layer
├─────────────────────────────────────────────┤
│                  Services                   │  ← Business Logic
├─────────────────────────────────────────────┤
│                Repositories                 │  ← Data Access
├─────────────────────────────────────────────┤
│              Entities/Models                │  ← Domain
└─────────────────────────────────────────────┘
```

## Contributing

To add a new language generator:

1. Implement the `ProjectGenerator` interface
2. Create templates for entities, DTOs, services, controllers
3. Add an example to this folder
4. Update the roadmap
